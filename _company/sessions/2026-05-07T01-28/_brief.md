[규칙 1] 증거 기반 작성 의무
모든 사실 진술은 아래 형식으로 출처를 명시해.
- COSLAB Hub 기능: [출처: COSLAB-HUB-DESIGN.md, 섹션 X]
- 개발 진행 상황: [출처: COSLAB-HUB-STATUS.md, 라인 Y]
- API 명세: [출처: COSLAB-HUB-API.md]
- DB 구조: [출처: COSLAB-HUB-SCHEMA.md]
- 코딩 규칙: [출처: COSLAB-HUB-CONVENTIONS.md]

문서에 없는 내용은 반드시 [추측] 또는 [확인 필요] 태그를 붙여.
태그 없이 단정적으로 말하면 안 돼.

[규칙 2] 금지 표현 (사용 시 작업 거부)
다음 표현은 절대 쓰지 마:
- "우리 부부의", "로맨틱한", "따뜻한 시선" (감성 마케팅 표현 금지)
- "획기적인", "혁신적인", "압도적인" (과장 형용사 금지)
- "맞춤형 가이드", "만능 키" (모호한 비유 금지)
- "A라는 일", "B라는 문제" (구체성 없는 placeholder 금지)
- 이모지를 보고서 본문에 쓰지 마 (제목/구분에만 허용)

[규칙 3] 수치/근거 의무화
모든 주장에는 수치 또는 근거를 붙여.
- 나쁜 예: "시간이 단축됩니다"
- 좋은 예: "[추측] 처방 등록부터 BPOM 제출까지 평균 90일이 60일로 단축 가능 (대웅 베타 데이터 확인 필요)"

[규칙 4] 대화 상대 고정
이 자료의 최종 독자는 두 명이야:
- 채우기 (B2B 영업 전문가): 시장 데이터, 가격, 경쟁 분석에 관심
- Sunny (시니어 풀스택 개발자, 477 커밋): 기술 깊이, DB 구조, 보안 구현에 관심

이 두 사람을 대상으로 작성해. "처음 듣는 사람" 대상으로 일반화하지 마.

[규칙 5] 1차 산출물은 항상 brainpack에 마크다운 저장
파일명: YYYY-MM-DD-[주제]-[버전].md
대화에서 산출물 본문을 길게 출력하지 말고, 핵심 요약 + 파일 경로만 알려줘.

SECTION 3 — 첫 임무: 5단계 순차 실행
═══════════════════════════════════════════

[STEP 1 — 30분 내] 문서 정독 보고
Researcher가 brainpack 폴더의 COSLAB-HUB-*.md 5개 파일을 읽고
각 파일별로 아래 정보를 추출해서 보고해줘:

1. COSLAB-HUB-DESIGN.md — 모듈 구조, 사용자 플로우, 핵심 기능
2. COSLAB-HUB-STATUS.md — 완성된 기능, 진행 중 기능, 미착수 기능 (% 단위)
3. COSLAB-HUB-API.md — 외부 노출 API 엔드포인트 수, 인증 방식
4. COSLAB-HUB-SCHEMA.md — 테이블 수, 핵심 엔터티 5개, 관계
5. COSLAB-HUB-CONVENTIONS.md — 코딩 표준, 보안 관련 규칙

각 파일당 5줄 이내 핵심 추출. 추측하지 말고 문서에 적힌 그대로.

[STEP 2 — STEP 1 완료 후] 경쟁사 실측 분석
Researcher가 https://cmstudioplus.com/ 에 실제 접속해서 아래를 조사해줘:
- 회사 정보 (본사, 설립연도)
- 제공 기능 (페이지에 명시된 모듈/기능 리스트)
- 가격 정책 (공개되어 있다면)
- 타겟 산업 (화장품 전용? 범용?)
- 지원 언어/지역
- 약점 (없는 기능, 인도네시아 미지원 등)

접속 불가 시 "확인 필요"로 표시하고 검색 가능한 공개 정보만 정리.
COSLAB Hub와의 기능 비교표를 만들어 (10개 항목 이상).

[STEP 3 — STEP 2 완료 후] 보안 강화 실행 계획
Researcher가 COSLAB-HUB-SCHEMA.md를 다시 읽고
Supabase RLS 적용을 위한 구체 계획을 작성해줘:
- 어떤 테이블에 RLS가 필요한지 (테이블명 명시)
- 각 테이블별 권한 정책 (예: tenant_id 기반 격리)
- 적용 우선순위 (Critical / High / Medium)
- 예상 작업량 (Sunny 공수 기준 일 단위)

이 단계는 100% 사실 기반. 추측 금지. 문서에 없으면 "Sunny 확인 필요" 표시.

[STEP 4 — STEP 3 완료 후] 가격 모델 시뮬레이션
Business가 SaaS 가격을 IDR 기준 3안으로 설계해줘.
각 안마다 아래 정보 필수:
- 월 구독료 (IDR + USD 환산, 환율은 1 USD = 16,000 IDR로 계산)
- 포함 모듈
- 사용자 수 제한
- 1년 매출 시뮬레이션 (고객사 5곳, 10곳, 20곳 시나리오)
- 가격 근거 (CM Studio Plus 또는 일반 SaaS 시장 기준)

[STEP 5 — 모든 단계 완료 후] CEO 종합 보고
CEO가 위 4개 산출물을 종합해서 한 페이지로 정리.
- 발견한 사실 5가지 (출처 포함)
- 즉시 액션 3가지 (담당자: 채우기/Sunny/공동)
- 의사결정 필요 사항 (있을 때만)
- 다음 주 진행 예정 작업
